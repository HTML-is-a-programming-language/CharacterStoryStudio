# Character Story Studio 작업 규칙

## 프로젝트 목적

이 프로젝트는 타인에이아이 AI-Native Developer 지원용 메인 포트폴리오다.

AI 캐릭터와 나눈 대화를 분석하여 사용자가 승인하고 수정할 수 있는
개인화 모션코믹 영상으로 변환하는 서비스를 만든다.

러비더비(타인에이아이의 AI 캐릭터 채팅 서비스)의 클론이 아니다. 러비더비는
공개 API가 없으므로 실제 서비스와 연동하지 않고, 독자적인 캐릭터/세계관을 사용한
자체 데모 채팅 UI로 대화를 만든다. OpenMontage(오픈소스 에이전트 기반 영상
제작 시스템)의 코드는 직접 의존성으로 쓰지 않는다. 단계별 승인 게이트를 둔
파이프라인 구조와 QA 검증 개념만 참고하여 TypeScript 스택으로 직접 구현한다.

## 작업 범위

- 현재 사용자가 승인한 Phase만 구현한다.
- 다음 Phase 기능을 미리 구현하지 않는다.
- 현재 저장소와 기존 파일을 확인한 뒤 작업한다.
- 여러 파일을 변경할 때는 먼저 계획을 제시한다.
- 기존 파일을 임의로 삭제하거나 전체 교체하지 않는다.

## 개발 환경

- Windows PowerShell에서 실행 가능한 명령을 우선한다.
- 패키지 매니저는 pnpm만 사용한다.
- TypeScript strict mode를 유지한다.
- 환경변수와 API 키를 코드에 작성하지 않는다.
- 서버 전용 코드가 클라이언트 번들에 포함되지 않게 한다.

## AI 생성 리소스 정책

- 이미지/영상/음성 등 유료 AI API 호출은 Mock Provider로 먼저 대체 가능하게 설계한다.
- API 키가 없어도 전체 흐름(Demo Mode)이 동작해야 한다.
- 비용이 발생하는 실제 Provider 호출은 사용자 승인 없이 자동 실행하지 않는다.

## 코드 작성 규칙

- UI, 도메인 로직, 외부 Provider 구현을 분리한다.
- React 컴포넌트에서 AI API를 직접 호출하지 않는다.
- 의미 없는 any 사용으로 타입 오류를 회피하지 않는다.
- 빈 catch 문으로 오류를 무시하지 않는다.
- 서로 다른 책임의 코드를 utils.ts 하나에 몰아넣지 않는다.
- 임시 구현에는 TODO와 현재 한계를 명확히 기록한다.
- 복잡한 코드보다 설명 가능한 단순한 구조를 우선한다.

## 품질 검증

변경 후 가능한 범위에서 다음을 실행한다.

- lint
- typecheck
- unit test
- build

실행하지 않은 검증을 통과했다고 보고하지 않는다.
실패한 명령은 오류 원인과 해결 여부를 명확히 보고한다.

## Git과 배포

- 요청 없이 git commit을 실행하지 않는다.
- 요청 없이 git push와 배포를 실행하지 않는다.
- 기존 사용자의 변경사항을 임의로 되돌리지 않는다.
- 강제 초기화와 대량 파일 삭제를 실행하지 않는다.

## 포트폴리오 문서 관리

- Phase(또는 의미 있는 작업 단위)가 끝날 때마다 `docs/PORTFOLIO.md`에 무엇을 했는지, 어떤 AI 도구/기능을 어떻게 썼는지, 작업 중 발생한 이슈와 대응을 기록한다.
- 설계/기술 판단이 있을 때마다 `docs/DECISIONS.md`에 ADR 형식(상태/배경/결정/결과)으로 근거를 남긴다.
- 두 문서는 코드와 별개로 항상 최신 상태를 유지한다 — "다음에 채워넣겠다"는 식으로 미루지 않는다.
- 대화가 끝날 때(또는 사용자가 요청할 때)마다 `docs/ASSISTANT_LOG.md`에 어시스턴트가 실제로 한 답변/작업만 순수 기록용으로 추가한다. 이 문서는 사용자의 개인적인 열람용이며, 이후 작업의 의사결정 근거로 참고하지 않는다.

## 작업 완료 보고

각 작업이 끝나면 다음을 보고한다.

1. 생성하거나 수정한 파일
2. 구현한 기능
3. 실행한 명령
4. 검증 결과
5. 남은 문제와 한계
6. 다음 권장 작업
